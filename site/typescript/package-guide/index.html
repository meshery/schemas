@meshery/schemas Package Guide

Installation, import paths, TypeScript type usage, and the complete deep-import reference for the Meshery schemas npm package.

Type Browser Package Guide RTK Hooks

Installation

npm install @meshery/schemas
# or
yarn add @meshery/schemas

The package ships pre-built TypeScript declarations. No additional build step is required in the consuming project.

Version compatibility

The package version tracks the schema repository version. Use the npm registry to find the latest release. The latest dist-tag always points to the most recent stable release.

Import patterns

Barrel import (top-level)

The package root re-exports all types from all constructs and all versions. Use the top-level import for most cases:

import type { Connection, ConnectionPayload, Environment } from "@meshery/schemas";

Deep import (per-construct)

For tree-shaking or when you need a specific version's types, import directly from the construct path:

import type { components } from "@meshery/schemas/typescript/generated/v1beta1/connection/Connection";

type Connection = components["schemas"]["Connection"];
Deep imports give you version-pinned types. Use the Type Browser to inspect available properties before importing.

Using types in TypeScript

Entity types (API responses)

import type { Connection, Environment, Workspace } from "@meshery/schemas";

async function getConnection(id: string): Promise<Connection> {
  const res = await fetch(`/api/integrations/connections/${id}`);
  return res.json() as Promise<Connection>;
}

Payload types (write operations)

Every writable entity has a corresponding *Payload type that contains only client-settable fields — no createdAt, updatedAt, or other server-generated fields.

import type { ConnectionPayload } from "@meshery/schemas";

const payload: ConnectionPayload = {
  name: "my-cluster",
  kind: "kubernetes",
  type: "In-Cluster",
  subType: "Kubernetes",
};

Type narrowing with discriminated unions

import type { Connection } from "@meshery/schemas";

function isConnected(c: Connection): boolean {
  return c.status === "connected";
}

TypeScript path configuration

If your tsconfig.json uses path aliases, map @meshery/schemas to its node_modules location:

{
  "compilerOptions": {
    "paths": {
      "@meshery/schemas": ["./node_modules/@meshery/schemas"],
      "@meshery/schemas/*": ["./node_modules/@meshery/schemas/*"]
    }
  }
}
The package's types and exports fields in package.json configure resolution automatically for modern bundlers (Vite, webpack 5, esbuild). Manual path config is only needed for very old setups.
