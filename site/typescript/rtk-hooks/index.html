RTK Query Hooks

Generated API hooks for React + Redux Toolkit — covering both Meshery Server and Layer5 Cloud endpoints.

Type Browser Package Guide RTK Hooks

What are the RTK Query hooks?

The build pipeline generates fully typed RTK Query API slices from the bundled OpenAPI specs. These hooks are the primary way Meshery's React UIs talk to their backends — no hand-written fetch calls, no manual type casting.

mesheryApi

Hooks for Meshery Server endpoints. Generated from _openapi_build/meshery_openapi.yml (operations tagged x-internal: ["meshery"]).

cloudApi

Hooks for Layer5 Cloud endpoints. Generated from _openapi_build/cloud_openapi.yml (operations tagged x-internal: ["cloud"]).

Installation

npm install @meshery/schemas @reduxjs/toolkit react-redux

Store setup

Register both API slices in your Redux store. Each slice manages its own cache, loading states, and invalidation logic automatically.

// store.ts
import { configureStore } from "@reduxjs/toolkit";
import { mesheryApi } from "@meshery/schemas/typescript/rtk/meshery";
import { cloudApi } from "@meshery/schemas/typescript/rtk/cloud";

export const store = configureStore({
  reducer: {
    [mesheryApi.reducerPath]: mesheryApi.reducer,
    [cloudApi.reducerPath]:   cloudApi.reducer,
  },
  middleware: (getDefault) =>
    getDefault()
      .concat(mesheryApi.middleware)
      .concat(cloudApi.middleware),
});

export type RootState = ReturnType<typeof store.getState>;
export type AppDispatch = typeof store.dispatch;

Using query hooks

Query hooks (generated from GET operations) follow the naming pattern use{OperationId}Query. The operationId in the schema (getConnections) becomes useGetConnectionsQuery.

import { useGetConnectionsQuery } from "@meshery/schemas/typescript/rtk/meshery";

function ConnectionsList() {
  const { data, isLoading, error } = useGetConnectionsQuery({
    page: 1,
    pageSize: 25,
  });

  if (isLoading) return <Spinner />;
  if (error)     return <ErrorBanner error={error} />;

  return (
    <ul>
      {data?.connections.map((c) => (
        <li key={c.id}>{c.name}</li>
      ))}
    </ul>
  );
}

Using mutation hooks

Mutation hooks (generated from POST, PUT, DELETE) follow the naming pattern use{OperationId}Mutation.

import {
  useCreateConnectionMutation,
  useDeleteConnectionMutation,
} from "@meshery/schemas/typescript/rtk/meshery";
import type { ConnectionPayload } from "@meshery/schemas";

function AddConnection() {
  const [createConnection, { isLoading }] = useCreateConnectionMutation();

  async function handleSubmit(payload: ConnectionPayload) {
    try {
      const result = await createConnection(payload).unwrap();
      console.log("Created:", result.id);
    } catch (err) {
      console.error("Failed:", err);
    }
  }

  // ...
}

Query argument shape

Query args map 1-to-1 with the OpenAPI operation parameters. All parameter names follow the camelCase schema contract — never snake_case.

// CORRECT — camelCase args matching the schema contract
useGetWorkspacesQuery({ orgId: "abc", pageSize: 10, page: 1 });

// WRONG — snake_case args (consumer audit will flag these as snake-case-param)
useGetWorkspacesQuery({ org_id: "abc", page_size: 10 });
The consumer audit CI job scans RTK Query sites in downstream repos and flags snake-case-param, case-flip, and snake-case-wrapper violations. Keep all arg/body keys in camelCase to pass the audit.

Cache invalidation

Tag-based cache invalidation keeps list queries fresh after mutations. Provide tags in the API slice or use invalidatesTags in your mutation endpoint:

// In a custom split API endpoint (advanced usage)
endpoints: (builder) => ({
  deleteConnection: builder.mutation({
    query: ({ connectionId }) => ({
      url: `/api/integrations/connections/${connectionId}`,
      method: "DELETE",
    }),
    invalidatesTags: ["Connection"],
  }),
})

Lazy queries

Use useLazy{OperationId}Query when you need to trigger a fetch imperatively (e.g. on button click or after a form submit):

import { useLazyGetConnectionsQuery } from "@meshery/schemas/typescript/rtk/meshery";

function SearchConnections() {
  const [trigger, { data, isFetching }] = useLazyGetConnectionsQuery();

  return (
    <button onClick={() => trigger({ search: "kube" })}>
      {isFetching ? "Searching…" : "Search"}
    </button>
  );
}
