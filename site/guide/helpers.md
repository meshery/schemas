# Helper Files

<p class="page-desc">
  <code>models/v1beta1/&lt;construct&gt;/&lt;construct&gt;.go</code> is auto-generated and overwritten on every <code>make build</code>. Put all hand-written Go — GORM overrides, SQL driver methods, Entity interface — in the sibling <code>*_helper.go</code> file.
</p>

Every helper file must start with:

```go
// This is not autogenerated.
package <construct>
```

That comment is the boundary between generated and manual code.

---

## TableName

Override when the GORM-inferred table name differs from the actual DB table:

```go
// This is not autogenerated.
package keychain

func (k *Keychain) TableName() string {
    return "keychains"
}
```

---

## Entity interface

```go
import "github.com/meshery/schemas/models/core"

func (k *Keychain) GetID() core.Uuid {
    return k.Id
}
```

## SQL driver — `Value()` and `Scan()`

Use `x-generate-db-helpers: true` on the schema component when the entity is stored as a JSON blob in one column — the generator emits `Value()`/`Scan()` automatically. For hand-written drivers (map-like types, etc.):

### `Value()` — always marshal

```go
func (m MapObject) Value() (driver.Value, error) {
    b, err := json.Marshal(m)
    if err != nil {
        return nil, fmt.Errorf("marshal MapObject: %w", err)
    }
    return string(b), nil // nil map marshals to JSON "null", not SQL NULL
}
```

Returning `(nil, nil)` writes SQL NULL. Don't do it unless the column is explicitly nullable and you've documented why.

### `Scan()` — zero the receiver on nil

```go
func (m *MapObject) Scan(src interface{}) error {
    switch v := src.(type) {
    case nil:
        *m = nil  // zero — avoids stale data if the struct is reused across rows
        return nil
    case string:
        return json.Unmarshal([]byte(v), m)
    case []byte:
        return json.Unmarshal(v, m)
    default:
        return fmt.Errorf("unsupported type: %T", src)
    }
}
```

## `x-generate-db-helpers`

| Situation | What to do |
|---|---|
| Entity stored as a single JSON blob column | `x-generate-db-helpers: true` at the schema **component** level |
| Entity mapped to a full DB table | Don't use it — implement `TableName()` and drivers by hand |
| Free-form JSON with no fixed schema | Use `x-go-type: "core.Map"` instead |

The annotation belongs on the component root, not on a property:

```yaml
# correct
TokenMetadata:
  x-generate-db-helpers: true
  type: object
  properties: ...

# wrong — on a property, has no effect
properties:
  metadata:
    x-generate-db-helpers: true
```

## Checklist

<ul class="checklist">
  <li>First line is <code>// This is not autogenerated.</code></li>
  <li><code>TableName()</code> set if the DB table name differs from the GORM default</li>
  <li><code>Value()</code> always marshals — never returns <code>(nil, nil)</code></li>
  <li><code>Scan()</code> zeroes the receiver (<code>*m = nil</code>) when <code>src</code> is nil</li>
  <li><code>x-generate-db-helpers: true</code> is on the component, not a property</li>
  <li><code>go test ./...</code> passes after adding the file</li>
</ul>
